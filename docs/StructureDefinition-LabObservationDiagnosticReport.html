Resource Profile: Lab Observation DiagnosticReport ( Abstract )

Official URL: http://va.gov/fhir/StructureDefinition/LabObservationDiagnosticReport Version: 1.5.11
Draft as of 2025-05-08 Computable Name: LabObservationDiagnosticReport

This StructureDefinition contains the maps for VistA file LABORATORY TEST (60) to us-core-diagnosticreport-lab.

​See also the FHIR Shorthand.

This is an abstract profile that is copied by other, concrete profiles. Each descendant concrete profile inherits the maps from this more general case. This way, we don’t have to maintain multiple copies of maps for closely related profiles. (Note that this inheritance is managed by the mapping team with a script; we’re not using the FHIR baseDefinition property for this.)

Sometimes, you will notice that an abstract profile includes only "target not supported" maps, so it doesn't look very useful. However, for every profile, we not only map the fields for which we find source data; we also stipulate "target not supported" for any unmapped fields that are tagged "must support" in the US Core. This way, implementers who might be concerned about these fields if they were not mentioned are fully informed about the reason they are not mapped. When there are several concrete profiles that require these clarifying maps, we collect them in an abstract parent, just as we would for cases where we have source element maps.

Use Case Overview

Usage:

  • CapabilityStatements using this Resource Profile: Lighthouse Patient Health API
  • This Resource Profile is not used by any profiles in this Implementation Guide
